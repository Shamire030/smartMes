# Smart MES项目分工计划（精简版）

## 1. 团队构成与职责（基于微服务架构）

### 1.1 团队分组与核心职责

#### 第一组：前端框架与核心页面开发
- **成员**：2人
- **职责**：前端项目搭建、核心页面开发、路由配置、UI组件库集成、权限管理
- **对应角色**：企业管理者、车间主任、生产计划员

#### 第二组：生产计划服务开发
- **成员**：2人
- **职责**：生产计划前后端功能开发、排产算法实现、ERP集成接口
- **对应微服务**：生产计划服务
- **对应角色**：生产计划员

#### 第三组：生产执行服务开发
- **成员**：2人
- **职责**：生产执行前后端功能开发、数据采集接口、实时数据处理、异常上报与通知
- **对应微服务**：生产执行服务
- **对应角色**：生产操作员、车间主任

#### 第四组：质量管理与设备管理服务开发
- **成员**：2人
- **职责**：质量管理前后端功能开发、设备状态监控功能开发
- **对应微服务**：质量管理服务、设备管理服务
- **对应角色**：质检员、设备管理员

#### 第五组：数据分析服务与系统集成
- **成员**：2人
- **职责**：总览看板开发、趋势分析图表开发、系统性能优化、数据库优化、集成测试
- **对应微服务**：数据分析服务
- **对应角色**：企业管理者、质量管理人员

### 1.2 成员具体分工明细

#### 第一组：前端框架与核心页面开发
- **成员A**：前端项目搭建、路由配置、组件库集成（Element Plus）、权限管理（JWT实现）
- **成员B**：核心页面布局、通用组件开发、ECharts图表集成、前端性能优化（页面加载<2秒）
- **验收标准**：前端架构完整，组件复用率≥80%，核心页面加载时间≤2秒

#### 第二组：生产计划服务开发
- **成员A**：生产计划后端API开发（Spring Boot+Spring Data JPA）、数据模型设计、ERP集成接口
- **成员B**：生产计划前端页面开发、拖拽式排程功能、订单管理功能
- **验收标准**：能从ERP接收订单，拖拽式排程功能正常，计划保存与发布功能可用

#### 第三组：生产执行服务开发
- **成员A**：生产执行后端API开发、数据采集接口、异常上报与通知机制（RabbitMQ）
- **成员B**：生产执行前端页面、扫码开工功能、产量/不良品记录、生产进度看板
- **验收标准**：扫码开工成功率100%，生产进度数据实时更新，异常通知及时送达

#### 第四组：质量管理与设备管理服务开发
- **成员A**：质量管理后端API开发（MyBatis）、质检数据录入功能、IQC/IPQC/FQC流程实现
- **成员B**：质量管理前端页面、设备状态监控页面、基础数据管理功能
- **验收标准**：质检数据录入完整准确，设备状态监控延迟<5秒

#### 第五组：数据分析服务与系统集成
- **成员A**：总览看板开发、趋势分析图表、数据库优化、Redis缓存集成
- **成员B**：系统集成、API文档编写（SpringDoc OpenAPI）、集成测试、性能测试
- **验收标准**：总览看板展示核心指标准确，系统并发支持≥500用户

## 3. 技术实现分工

### 3.1 前端技术分工
- **第一组**：Vue 3 + Vite + Element Plus + ECharts + JWT权限管理
- **第二组**：生产计划页面 + 拖拽组件 + 订单管理页面
- **第三组**：生产执行页面 + 扫码功能 + 进度看板
- **第四组**：质量管理页面 + 设备监控页面 + 基础数据管理页面
- **第五组**：数据分析看板 + 趋势图表 + 性能优化

### 3.2 后端技术分工
- **第二组**：Spring Boot + Spring Data JPA + ERP集成接口
- **第三组**：Spring Boot + RabbitMQ + 数据采集接口
- **第四组**：Spring Boot + MyBatis + 质量管理服务 + 设备管理服务
- **第五组**：Spring Boot + Redis + 数据分析服务 + 系统集成

---

**备注**：本分工计划基于10人团队、两周开发周期设计，采用微服务架构模式，确保各团队成员职责明确，工作内容完整覆盖项目需求。
